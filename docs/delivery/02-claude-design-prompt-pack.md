# Claude Design Prompt Pack

Use this pack before any major visual redesign pass.

## Product framing

Design a premium pickleball club web app with five surfaces:

1. Marketing + landing
2. Public booking and checkout
3. Member account and booking control
4. Admin live operations control room
5. External TV liveboard

The product is not just a booking app. It must handle the full state journey:

`discover -> reserve -> pay -> arrive -> check in -> queue -> play -> time up -> next group`

## Visual direction

- Avoid generic SaaS gradients on white.
- Use a dark athletic-broadcast atmosphere for admin and liveboard.
- Use a cleaner but still premium version of that system for public booking.
- Typography should feel intentional and energetic, not corporate-default.
- Large-screen readability matters as much as mobile polish.

## Non-negotiable UX priorities

- Mobile-first public booking.
- Guest and member flows both first-class.
- Payment clarity before commitment.
- Booking status easy to understand after payment.
- Admin can scan the floor fast.
- TV board readable from distance with big hierarchy and strict restraint.

## Responsive rules

- Phone:
  - booking first
  - one clear primary action per section
  - short summaries, no dense tables
- Tablet:
  - staff-friendly live control
  - split panes allowed
- Desktop:
  - admin information density increases
- TV:
  - extreme readability
  - no tiny text
  - no sensitive information

## Tone by surface

- Landing: confident, premium, conversion-focused
- Booking: calm, clear, low-friction
- Account: reassuring and status-oriented
- Admin: operational, compact, high-trust
- TV: broadcast-like, alive, obvious
