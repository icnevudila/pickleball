# Reference Analysis

## Available Reference

The visible reference in the conversation was a dark-mode interface screenshot with a modern social/dashboard feeling.

No external URL links were visible in the chat context. This document therefore analyzes the uploaded visual direction and the product requirements already defined.

## Observed Direction From Reference

The visual direction suggests:

- Dark UI
- High contrast panels
- Card-based layout
- Strong hierarchy
- Realtime/live feeling
- Dense but readable content
- Sidebar/topbar navigation pattern
- Modern SaaS/social interface style

## What Fits This Product

### Good Ideas To Keep

- Dark mode for liveboard and admin live operation
- Strong card layout
- Status badges
- Avatar-heavy UI
- Compact but scannable information
- Clear separation between active/queued/available states

### Things To Avoid

- Too much small text on TV
- Social-feed style clutter
- Low contrast gray-on-black text
- Too many controls on one screen
- Fancy animation that hurts readability
- Making TV screen look like admin screen

## Design Translation

The product needs three visual layers:

### 1. Player Layer

Should feel simple, friendly, mobile-first.

Mood:

```txt
Book fast, understand status, leave.
```

### 2. Admin Layer

Should feel like a control room.

Mood:

```txt
See everything, act fast, recover mistakes.
```

### 3. TV Liveboard Layer

Should feel like a sports broadcast scoreboard.

Mood:

```txt
Big, alive, readable, exciting.
```

## Key UI Decision

Use dark theme for:

```txt
Admin liveboard
TV liveboard
Design system preview
```

Use clean mixed/light sections for:

```txt
Player booking
Profile
Checkout
```

A full dark player UI is acceptable, but booking conversion often benefits from clearer cards and less visual noise.

## Product-Specific Design Requirement

This is not just a booking app.

The design must show state transitions clearly:

```txt
Booked
↓
Checked in
↓
Waiting
↓
Next up
↓
Playing
↓
Time up
↓
Completed
```

If those states are not visually obvious, the UI is not doing its job.
