## Product

Pickleball Booking & Liveboard Management System

## Design Goal

Create a product that feels like a modern sports club command center:

```txt
Players book fast.
Admins operate under pressure.
TV liveboard looks alive, readable, and premium.
```

## Recommended Design Direction

The system should combine:

- Sports broadcast energy
- SaaS dashboard clarity
- TV scoreboard readability
- Mobile-first booking simplicity
- Admin control-panel practicality

## MVP Design Priorities

| Priority | Area | Reason |
|---|---|---|
| 1 | TV liveboard readability | People view it from distance |
| 2 | Admin speed | Staff will use it during live sessions |
| 3 | Booking clarity | Players should book without thinking |
| 4 | State visibility | Everyone must know what is happening |
| 5 | Realtime confidence | Users should trust the screen is live |

## Design Principle

Pretty is nice. Clear wins.

The product can look slick, but if a player cannot tell when they play next, the design failed.
