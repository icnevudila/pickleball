# Design QA Checklist

## Player Booking

- [ ] Session availability is visible without opening detail.
- [ ] Book button is obvious.
- [ ] Full/waitlist state is clear.
- [ ] Payment status is clear.
- [ ] Booking confirmation includes date/time.
- [ ] Mobile layout is clean.
- [ ] Error messages are human-readable.

## Admin Dashboard

- [ ] Active session is easy to find.
- [ ] Open liveboard button is prominent.
- [ ] Unpaid bookings are visible.
- [ ] Quick check-in is available.
- [ ] Tables are readable.

## Admin Liveboard

- [ ] Court statuses are obvious.
- [ ] Current players are visible.
- [ ] Timer is visible.
- [ ] Queue can be managed.
- [ ] Next-up group is clear.
- [ ] Start/end game actions are obvious.
- [ ] Dangerous actions require confirmation.
- [ ] Admin can recover from mistakes.

## TV Liveboard

- [ ] Readable from distance.
- [ ] Timer is very large.
- [ ] Player names are readable.
- [ ] Player photos/initials are clear.
- [ ] Court available state is obvious.
- [ ] Next-up group is obvious.
- [ ] Waiting queue does not clutter the screen.
- [ ] Reconnecting state exists.

## Visual Design

- [ ] Color usage is consistent.
- [ ] Status badges use text and color.
- [ ] Spacing is consistent.
- [ ] Cards are visually grouped.
- [ ] Buttons have clear hierarchy.
- [ ] Dark mode contrast is strong.

## Accessibility

- [ ] Buttons have labels.
- [ ] Inputs have labels.
- [ ] Focus states exist.
- [ ] Text contrast is acceptable.
- [ ] UI does not rely only on color.
- [ ] Reduced motion is considered.

## Handoff

- [ ] Components are named.
- [ ] States are documented.
- [ ] Empty states included.
- [ ] Loading states included.
- [ ] Error states included.
- [ ] Responsive layouts included.
