# Microcopy and Content Guide

## Tone

The product voice should be:

```txt
Clear
Direct
Friendly
Sporty
Calm under pressure
```

Avoid confusing technical terms for players.

## Player Copy

### Booking CTA

Use:

```txt
Reserve spot
Join waitlist
Pay and confirm
View booking
Cancel booking
```

Avoid:

```txt
Create participant record
Submit session transaction
```

### Booking Confirmation

```txt
You’re booked.
Arrive 10 minutes early and check in at the desk.
```

### Payment Pending

```txt
Payment is pending. Your booking is not confirmed yet.
```

### Payment Manual

```txt
Your spot is reserved. Payment will be collected at the club.
```

### Waitlist

```txt
This session is full. Join the waitlist and we’ll notify you if a spot opens.
```

## Admin Copy

### Check-in

```txt
Check in
Mark no-show
Add to queue
```

### Court Actions

```txt
Assign players
Start game
End game
Extend +5 min
Set maintenance
```

### Confirmations

```txt
End this game?
Players will be removed from the court and the court will become available.
```

```txt
Remove player from queue?
You can add them back later if needed.
```

### Error Copy

```txt
Court is no longer available. Liveboard has been refreshed.
```

```txt
Not enough waiting players to build a doubles group.
```

## TV Copy

Keep TV copy short.

Good:

```txt
LIVE
PLAYING
AVAILABLE
NEXT UP
TIME UP
WAITING
```

Avoid long messages:

```txt
The current court assignment has exceeded its originally configured duration.
```

Use:

```txt
TIME UP
Waiting for admin
```

## Content Rule

Short copy wins during live operations.

Admins do not read paragraphs while players are waiting.
