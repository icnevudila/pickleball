# Acceptance Criteria

1. Sayfalar Live Desk ailesinden kopmamalı: cream/clay/lime palette, güçlü action rail, status badges, büyük okunabilir typography.
2. Büyük koyu kahverengi bloklar ana yüzey olarak kullanılmamalı; koyu ton sadece marka, TV, terminal veya güçlü CTA yerlerinde kullanılmalı.
3. Her sayfanın ana görevi ilk 3 saniyede anlaşılmalı.
4. Admin ekranları tablo tarama ekranı gibi değil, aksiyon/komuta ekranı gibi davranmalı.
5. Booking ekranları takvim/kort/saat eksenini göstermeli; liste yardımcı olmalı.
6. Payment, split, wallet, cancellation policy gibi riskli durumlar badge + next action ile görünmeli.
7. Uzun isimler, boş state, çok kayıt, hata state ve loading state için ayrı UI sağlanmalı.
8. Kiosk ve TV ekranları desktop admin layout kullanmamalı; cihaz karakteri korunmalı.
9. Tailwind componentleri HTML yapısını kör kopyalamamalı; component sistemine ayrıştırılmalı.
