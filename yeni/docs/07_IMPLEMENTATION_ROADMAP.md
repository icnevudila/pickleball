# 07 — Implementation Roadmap

## Phase 0 — Prototype alignment

Goal: validate visual language and product scope.

Deliverables:

- Landing direction
- Admin command center mock
- Public booking mock
- Queue mock
- Court management mock
- Mobile and web screen set

## Phase 1 — MVP foundation

Goal: usable by one pilot club.

Build:

- Auth and club setup
- Courts CRUD
- Opening hours and pricing
- Public booking URL
- Availability engine
- Reservation creation
- Admin reservation calendar
- Basic live court status
- Manual payment status

Do not overbuild:

- Complex ranking
- Multi-location accounting
- Advanced coach marketplace
- AI scheduling

## Phase 2 — Live operations

Goal: replace the front desk spreadsheet/WhatsApp workflow.

Build:

- Command center
- Queue/waiting list
- Queue offers and expiration
- Court countdown
- Liveboard display
- Staff audit logs
- Notification templates

## Phase 3 — Payments

Goal: reduce no-shows and unpaid bookings.

Build:

- Deposits
- Payment Intent/Checkout integration
- Webhooks
- Refund rules
- Payment dashboard
- Failed payment recovery

## Phase 4 — Community layer

Goal: increase repeat usage.

Build:

- Player profiles
- Open matches
- Match result/activity feed
- Club leaderboard
- Weekly challenges
- Player tags and levels

## Phase 5 — Scale features

Build:

- Multi-club organization support
- Advanced reporting
- Coach booking
- Membership packages
- Dynamic pricing
- Integrations with WhatsApp/SMS/email
- TV liveboard mode

## MVP success metrics

- Booking completion rate
- Court utilization rate
- Queue conversion rate
- No-show rate
- Deposit/payment completion rate
- Manual staff interventions per day
- Repeat player bookings
